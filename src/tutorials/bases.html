Base conversion

Numeric bases are a way of representing numbers. We often represent numbers in decimal, which is base ten ("deci" means ten). However, we can represent these numbers in other numeric bases, too. For example, computers usually represent numbers in binary, which is base two. It's still the same number, but our way of representing it is different. This difference in representation is important because certain bases are useful for different tasks, as this tutorial will explain.

An introduction to numeric bases

As mentioned, decimal is the most commonly used base in modern society. The ancient Babylonians, for example, likes base 60, which is why the number 60 is used in time keeping (60 seconds in a minute, 60 minutes in an hour, etc). Ten seems like a natural base for humans because we have ten fingers.

A base is the number at which we move to the next digit place. In base ten, we have one digit for numbers one through nine. Once we hit ten, we now have two digits. Each digit in a number has a value such that the base is raised to a subsequentially larger power:

This works for all bases. Binary, for example, is base two. There's only two possible digits: 0 and 1. So naturally, all binary numbers are composed solely of zeros and ones. Since 1 is the largest value a binary digit can have, incrementing that would require us to move to the next digit place. So just like how adding 110 to 910 gives us 1010, adding 12 to 12 gives us 102.

In this tutorial, we will disambiguate between the different bases with a subscript. After all, "10" in decimal means something entirely different than it does in binary. Thus, 1010 refers to 10 in decimal, while 102 refers to 10 in binary.

Binary is a common base because most modern computers use binary to store data. All data on computers is stored as numbers, and it is up to the software and hardware to interpret these numbers as something meaningful. This actually is a lot like numeric bases. The number "10" could refer to very different numbers depending on what base we are using. Binary is the dominant choice for computers because it's so easy to represent in a machine. For our decimal system, we have ten different digits. Machines instead use two different voltage levels. So a high voltage can be considered a 1 and a low voltage can be considered a 0. Thus, we could store numbers as electricity (other methods include magnetism, as hard drives do, or small grooves, as CDs do).

Hexadecimal, which is base 16, is another commonly used base. It can fit exactly four binary digits in a single hexadecimal digit. Now that we can convert numbers, we can verify that 11112 (the largest possible four digit binary number) equals the largest one digit hexadecimal number, which has a value of 15.

Since hexadecimal has 16 different digits, we need some way to represent them visually, just like how we have the glyph "3" to represent the binary number three. For hexadecimal, we use the first five letters of the latin alphabet to represent the numbers ten through fifteen. So A16 equals 1010, B16 equals 1110, etc.

The reason hexadecimal is so popular is not only because it fits exactly four binary digits inside a single hexadecimal digit (making hexadecimal numbers much shorter and more readable), but most modern computers use processors that have a word size of 32 or 64 bits. A word is the natural amount of data that the processor likes to work with. We won't go into detail about processors here, but the bottom line is that binary data tends to have a number of bits which is a multiple of 32.

Sinze we can fit four bits ("binary digits") in a hexadecimal digit, each hexadecimal digit can store up to 24 numbers. On the other hand, a 32 bit number can store 232 numbers. Converting decimal to other bases

To convert decimal to other bases, we essentially perform this same formula in reverse. One method of doing this is dividing our base into the decimal number repeatively. In this, we perform integer division until we get to zero, and the remainders tell us our number, in reverse. This is best explained with an example. Let's convert 24 to binary:

24 / 2 = 12 remainder = 0
12 / 2 = 6 remainder = 0
6 / 2 = 3 remainder = 0
3 / 2 = 1 remainder = 1
1 / 2 = 0 remainder = 1

So we just keep dividing until we get to zero. Recall that integer division truncates any decimal, so 3 / 2 = 1. Anyway, reading the remainder in reverse is our binary number. Thus, 2410 = 110002.

This works for all bases. Thus, we could convert 24510 to hexadecimal:

245 / 16 = 15 remainder = 5
15 / 16 = 0 remainder = 15

Reading the remainders from bottom to top and converting to letters where needed, we get 24510 = F516.
